# 像素方块地图编辑器 — 设计文档

## 概述

一个纯前端像素风格地图编辑器，单 HTML 文件，浏览器打开即用。支持绘制墙体、放置角色、操控角色移动、导出 JSON 地图数据。

## 技术选型

- **纯 Canvas 渲染**：网格、墙体、角色全部在 `<canvas>` 上以像素方块绘制
- **单 HTML 文件**：内联 CSS + JS，零依赖，无框架
- **16×16 格子**：默认尺寸，后续可改为可配置

## 架构

```
map-editor.html
├── <canvas>          ← 地图绘制 + 角色显示
├── <style>           ← 像素风格 UI 样式
└── <script>          ← 全部逻辑
    ├── 状态管理      ← 网格数据、编辑器模式、角色位置
    ├── 渲染器        ← Canvas 绘图函数
    ├── 事件处理      ← 鼠标点击/拖拽编辑、键盘移动角色
    └── 导出          ← JSON 格式
```

### 状态层

集中管理的变量：

| 变量 | 类型 | 说明 |
|------|------|------|
| `grid[row][col]` | `boolean[][]` | `true` = 墙体，`false` = 空地 |
| `mode` | `enum` | `'wall'` / `'erase'` / `'place_hero'` / `'play'` |
| `hero` | `{row, col}` \| `null` | 角色当前格子位置 |
| `tileSize` | `number` | 每个格子的像素大小（根据画布大小自动计算） |

### 渲染层

每一帧做：
1. `ctx.clearRect()` 清空
2. 绘制网格线（浅灰色细线）
3. 遍历 `grid`，墙体格子填充砖块纹理（深灰 + 砖缝线）
4. 如果角色存在，画出像素小人（含简单的像素动画帧）
5. 如果当前是放置角色模式，鼠标悬停格子高亮（绿=可行，红=墙体上不可放）

### 编辑工作流

| 模式 | 操作 | 效果 |
|------|------|------|
| 画墙 | 点击/拖拽格子 | 放置墙体 |
| 擦除 | 点击/拖拽格子 | 清除墙体 |
| 放置角色 | 点击空地 | 角色出现在该格 |
| 操控 | 方向键/WASD | 角色移动（不可穿墙） |

工具栏按钮切换模式，有视觉高亮表明当前模式。

### 导出

JSON 格式：
```json
{
  "width": 16,
  "height": 16,
  "tiles": [[0, 1, ...], ...],   // 0=空地, 1=墙体
  "heroStart": { "row": 7, "col": 7 }
}
```
点击"导出"按钮触发浏览器下载 `.json` 文件。

### 地形预留

目前只有墙体和空地两种地形，但 `grid` 存储为数字（0, 1, ...）而非布尔，后续可扩展为 0=空地, 1=墙体, 2=水 等。

## 像素风格细节

- 格子上绘制简单的砖块纹理：深灰色底色 + 浅灰砖缝线
- 角色为像素小人的极简设计（约 4-6 个彩色方块）
- 工具栏按钮用像素字体（CSS `image-rendering: pixelated`）
- 整体配色偏复古：深色背景 + 亮色边框按钮
