# 斜角墙（Diagonal Wall）设计

## 概述

在现有墙体系统上增加斜角墙：一种在四个角中有一个缺口、允许球从该缺口对角通过的墙体。斜角墙是普通墙的子类 —— grid 中仍是 `T_WALL`，额外用 `diagCorners` Map 记录缺口朝向。

## 数据模型

```javascript
// grid 不变：斜角墙也是 T_WALL = 1
const diagCorners = new Map();  // "row,col" → 'TL' | 'TR' | 'BL' | 'BR'
```

四种缺口朝向：

| 朝向 | 含义 |
|------|------|
| `TL` | 左上角缺口 |
| `TR` | 右上角缺口 |
| `BL` | 左下角缺口 |
| `BR` | 右下角缺口 |

## 拐角→缺口映射

球对角移动时需要检查两个拐角格，各需特定朝向的缺口才放行：

| 球方向 | c1 拐角格需缺口 | c2 拐角格需缺口 |
|--------|:-----------:|:-----------:|
| ↖ (-1,-1) | BL | TR |
| ↗ (-1,+1) | BR | TL |
| ↙ (+1,-1) | TL | BR |
| ↘ (+1,+1) | TR | BL |

## 行为规则

| 移动类型 | 斜角墙行为 |
|---------|-----------|
| 角色（任意方向） | ❌ 阻挡（与普通墙一致） |
| 球正交移动 | ❌ 阻挡（与普通墙一致） |
| 球对角移动 — 拐角格缺口不匹配 | ❌ 阻挡 |
| 球对角移动 — 拐角格缺口匹配 | ✅ 放行 |

## 实现要点

### 无需改动的部分

- `tryMoveHero` — 斜角墙就是 `T_WALL`，自动阻挡
- `getPushChain` — 同上
- `isSolid` — 同上
- `placeHero` / `placeBall` / `placeCrate` — 同上

### 需改动的部分

1. **`followBall` 拐角检测**：在 `c1clear` / `c2clear` 判定中增加斜角墙缺口匹配分支
2. **绘制**：新增 `drawDiagWallTile(row, col, corner)` — 普通墙体 + 缺口指示
3. **编辑器 UI**：斜角墙按钮（循环切换 4 种朝向）+ 放置/擦除逻辑
4. **渲染**：`render()` 中对 `diagCorners` 中的格子调用 `drawDiagWallTile`
5. **导出/清空**：导出时编码缺口信息，清空时清除 `diagCorners`
6. **键盘快捷键**：`[7]` 激活斜角墙模式
